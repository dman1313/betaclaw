# 🧠 Memory Structure

## Files
- `MEMORY.md` — Long-term curated memory (preferences, decisions, lessons)
- `memory/YYYY-MM-DD.md` — Daily raw logs
- `memory/projects/` — Per-project state files
- `memory/preferences.md` — Dwayne's preferences and patterns
- `memory/failures.md` — Mistakes turned into learning signals

## End-of-Session Protocol
After meaningful sessions:
1. Extract new skills → write to `skills/` as SKILL.md files
2. Update project state in `memory/projects/`
3. Log preferences discovered in `memory/preferences.md`
4. Document failures in `memory/failures.md`
5. Update `MEMORY.md` with anything worth keeping long-term

## Heartbeat Maintenance
During idle heartbeats:
- Consolidate daily files into MEMORY.md
- Review failures and extract lessons
- Organize skills directory
- Check project states for stale items
