# 2026-04-07 - BEEFree Newsletter Designer Setup

## Installed Components

### BEEFree SDK Integration
- **Location**: `~/workspace/beefree/`
- **Skill**: `~/workspace/skills/beefree/SKILL.md`

### Files Created

1. **auth-proxy.js** - Server-side authentication proxy
   - Secure token generation (never exposes credentials to frontend)
   - Endpoints: `/proxy/bee-auth`, `/proxy/bee-refresh`, `/health`
   - Auto-refresh handling for 12-hour token lifecycle

2. **catalog-client.js** - Template catalog API client
   - Search and browse 2000+ BEEFree templates
   - CLI interface: `node catalog-client.js list|get|search`
   - NPM scripts: `npm run catalog:*`

3. **newsletter-editor.html** - Visual newsletter editor UI
   - Full BEEFree SDK integration
   - Drag-and-drop editor with save/export
   - JavaScript API: `window.beeEditor.init()`, `.save()`, `.export()`

4. **package.json** - Dependencies and scripts
   - Dependencies: `@beefree.io/sdk`, `express`, `cors`, `axios`, `dotenv`
   - Scripts: `start`, `catalog:*`

5. **Configuration**
   - `.env.example` - Template for credentials
   - `.gitignore` - Security (never commit credentials)

## Setup Requirements

To use the BEEFree editor, you need:

1. **BEEFree Developer Account**: https://developers.beefree.io/signup
2. **Application credentials**: Client ID and Client Secret
3. **Add to `.env`**:
   ```env
   BEEFREE_CLIENT_ID=your_id
   BEEFREE_CLIENT_SECRET=your_secret
   ```

## Usage

### Start Auth Proxy
```bash
cd ~/workspace/beefree
npm start
# Runs on http://localhost:3001
```

### Open Editor
```bash
cd ~/workspace/beefree
python -m http.server 8080
# Visit: http://localhost:8080/newsletter-editor.html
```

### Template Catalog
```bash
# Search templates
npm run catalog:search newsletter marketing

# Get specific template
npm run catalog:get template-slug
```

## Technical Notes

- **DNS Issue**: `catalog.getbee.io` may not resolve in sandboxed environments (expected)
- **Security**: All authentication is server-side; credentials never reach frontend
- **Token Lifecycle**: 12-hour sessions with auto-refresh
- **Error Handling**: Graceful recovery from token expiration (codes 5101, 5102)
